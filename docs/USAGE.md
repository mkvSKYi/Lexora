# Using Lexora

A quick guide to everything the app does. Lexora works fully offline.

## Add a book

Open the app and tap the **+** button on the library, then pick an `.epub` file.
Imported books appear on the shelf with a generated cover and your reading
progress. The most recently opened book sits at the top as **Continue reading**.

Long-press a book for **Details** or **Delete**.

## Read

Tap a book to open it. Swipe horizontally to turn pages. Tap the top edge to show
or hide the toolbar; it auto-hides while you read.

The bottom bar shows the chapter and a scrubber — drag it to jump anywhere in the
book. The list icon opens the table of contents.

## Translate a word or sentence

- **Tap a word** → a card with its IPA, part of speech, English definitions, and a
  Ukrainian translation. If the word isn't in the dictionary, an offline machine
  translation fills in.
- **Long-press** → translate the whole sentence.
- Tap **Save** on the card to add the word to your vocabulary.

## Vocabulary

Open **Saved words** from the bookmark icon on the library.

- Tap the **check** on a card to mark a word *learned* — it turns green and the
  progress bar at the top updates.
- Filter by **All / Learning / Learned**.
- Tap any saved word to see its full dictionary entry again.
- Delete with the trash icon.

## Make it yours

Open a book and tap **Aa** in the toolbar:

- **Theme** — Light, Sepia, Dark, AMOLED, Paper, Nord, Solarized Dark, Gruvbox, Dusk.
- **Font** — Default, Serif, Sans, Literata, Lora, Atkinson Hyperlegible, Inter,
  OpenDyslexic. Each chip previews in its real typeface.
- **Font size, line spacing, margins, page mode** (paginated or scrolling).
- **Brightness** and **warmth** sliders.

Changes apply as you turn the next page.

### Brightness gesture

Swipe **up/down along the right edge** of the page to brighten or dim — like a
video player. It won't interfere with tapping words or turning pages.

## Build from source

Requirements: Android Studio (JDK 21), Android SDK (compileSdk 36), a device or
emulator on API 26+.

```bash
./gradlew :app:installDebug      # build and install
./gradlew testDebugUnitTest      # run unit tests
```

The offline dictionary database is generated by a dev-time script in
`tools/dictionary/` from a [kaikki.org](https://kaikki.org) Wiktextract extract;
the prebuilt database ships in the app, so you don't need to run it.

### Release build

Release builds are minified (R8 + resource shrinking) and split per ABI — the
arm64-v8a APK is ~28 MB:

```bash
./gradlew :app:assembleRelease   # outputs app/build/outputs/apk/release/*.apk
```

To sign release builds, create a keystore and a gitignored `keystore.properties`
at the repo root; without it, release builds fall back to debug signing.

```bash
keytool -genkeypair -v -keystore release.jks -alias lexora \
  -keyalg RSA -keysize 2048 -validity 10000
cat > keystore.properties <<'EOF'
storeFile=release.jks
storePassword=YOUR_PASSWORD
keyAlias=lexora
keyPassword=YOUR_PASSWORD
EOF
```
